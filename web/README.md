# Patra Darpan (Mirror)

A high-performance, single-page application for browsing the Indian Journal of History of Science (IJHS) archives.

> **Darpan** (Sanskrit/Hindi) means "Mirror". This project mirrors the INSA archives with a modern, reflection-like instant interface.

## 🚀 Features

- **Instant Search**: Fuzzy search across 2000+ papers.
- **Hybrid Link Technology**:
  - **Local Mode**: Loads PDFs instantly from your local disk (`assets/pdfs` symlink).
  - **Cloud Mode**: Redirects to the official INSA server when hosted on Netlify (saving 1GB of bandwidth/storage).
- **Glassmorphism UI**: Premium aesthetic with dark mode by default.

## 🛠️ Local Setup

1. **Prerequisites**: Ensure you have the `ijhs-potentials` or `cached_papers` corpus in your `assets` folder.
2. **Generate Data**:
   Run the build script to index the TSV and create the symlinks.

   ```bash
   # From cahc-utils/scrape/
   uv run --with pandas tools/build_darpan_data.py
   ```

   _This creates `assets/js/data.js` and verifies the `assets/pdfs` symlink._

3. **Run Locally**:
   Open `index.html` directly in your browser, or serve it:
   ```bash
   python -m http.server
   ```
   _The app automatically detects it is running locally and serves local PDFs._

## ☁️ Deployment Workflow

We use **Netlify CLI** for a manual deployment strategy to avoid uploading the 1GB+ PDF corpus to Git.

### 1. Test Locally (`netlify dev`)

Simulates the Netlify environment locally.

```bash
netlify dev
```

- access at `http://localhost:8888`
- App label should show **"Netlify (Local)"** or similar, treating it as a cloud env.

### 2. Partial/Draft Deploy (`deploy`)

Push a private draft to a unique URL. Use this to share with the team for review before going live.

```bash
netlify deploy
```

- **Output**: A "Website Draft URL" (e.g., `https://draft-id--patra-darpan.netlify.app`).
- **Note**: The `assets/pdfs` folder is **ignored** (via `.netlifyignore`) to keep uploads fast and small (< 1MB).
- **Hybrid Mode**: The app will detect it's on Netlify and serve PDFs from the official INSA server.

### 3. Production Deploy (`deploy --prod`)

Push the changes to the main live URL.

```bash
netlify deploy --prod
```

- Updates the main site immediately.

## ⚙️ Configuration

- **`netlify.toml`**: Configures build settings and headers.
- **`.netlifyignore`**: Ensures `assets/pdfs` is NEVER uploaded.

## 🏗️ Architecture

- **`index.html`**: The single page shell. Detects hostname to switch modes.
- **`assets/js/data.js`**: Pre-computed database of papers. Generated by `tools/build_darpan_data.py`.
- **`assets/pdfs`**: A symbolic link to your local PDF corpus. Ignored by git and Netlify.
